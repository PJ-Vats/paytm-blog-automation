Where the friction was

Manual doc creation: content had to be copied into a newly created Drive doc, saved, and linked.

Manual roadmap maintenance: each blog required repeated updates to title, doc link, publish URL, and date.

Image effort stayed separate: image generation could still take multiple attempts, whether done directly or through App Script.

What was already strong

Bulk publishing already worked: 20–25 blogs could run in a single cycle.

Publishing automation existed: content creation and WordPress publishing were already connected.

Tracking discipline existed: publish-date visibility had already been added to the roadmap.

What makes the current phase strong

Auto-skip intelligence: if a title fails sanity checks, the workflow advances instead of stalling.

Live dashboard visibility: the sidebar shows the stage, progress, logs, and processed / published / skipped counts in real time.

Web submission path: titles can also be sent through a dedicated web view using the same backend logic.

Human review still matters: manual approval is preserved where editorial confidence matters most.

Current bottleneck

Image generation quality: multiple retries may still be needed before the image matches expectations.

Throughput impact: image iteration is the biggest reason a blog may take up to five minutes.

Opportunity: improving prompt reliability here is the clearest way to speed up the end-to-end system.
